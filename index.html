Understanding User Feedback

Collecting Feedback

User feedback can be collected through multiple methods such as reviews, in-app surveys and emailed feedback forms. Approaching the user feedback in multiple methods allows for more results and better analysis of the data, rather than waiting for user reviews to come in, set out a list of questions specific to multiple areas for a better understanding of where the focus needs to be shifted towards.

Analysing Feedback

The data received needs to be categorised into common themes such as bugs, feature requests, user experience issues etc. The data is best represented graphically to allow for easier interpretation for other colleagues working at the company. With the data collected, the company should prioritise feedback based on the frequency and impact it has on user experience. For example, if there is a high frequency of users saying their notifications aren’t coming through, this should be something that needs to be addressed quickly.

Resource Allocation

Software Engineering

The software engineers are tasked with two main categories which are maintenance and new features. Resources need to be appropriately designated to each area depending on the current situation regarding the app. For example, if the app is currently receiving a lot of negative feedback because of bugs, more work should be done on fixing it rather than developing new features. While the app has little to no bugs, then the resources should be shifted more towards new feature design and implementation.

UX Design

Similarly, with software development, the people who are designing the user interface and improving the UX should reflect on the current situation regarding user feedback.

Product Management

As a team, it’s important to review the progress made and adjust priorities therefore weekly meetings should occur to check in on the team and where they are at with the assigned work. It is also important to adjust emerging issues so more meetings may be scheduled if necessary.

Professional Development

Agile Development

Development should be broken down into short, repeatable phases that are a set length. Common phases include planning, designing, developing, testing, deploying, reviewing and launching. A suitable time frame to complete all these phases is usually 4 weeks and can be great for working on new features especially when the team consists of a small group of members because it means the focus is always on what’s needed.

Kanban Board

Outside of weekly meetings, it's still important to see what tasks are completed and what is left to be done. A Kanban board allows users to update tasks into categories and assign them to certain members. Each task can also include subtasks and overall provide a visual representation of the development objectives.

Metrics

To measure if the work done is effective, metrics and indicators should be measured and reviewed to see if the changes and new features impact the user engagement of the application. The best metrics to employ are MAU (monthly active users), churn rate and user ratings.

Objective Roadmap

  • 1. User Feedback

    In the first two months, we will gather user feedback through surveys, reviews, interviews, and focus groups. This will help identify common themes, prioritise issues, and understand user needs, guiding future improvements and feature developments.

  • 2. Quick Wins

    We'll tackle high-priority bugs and usability issues in the first two months to improve user experience, reduce churn, and stabilise Monthly Active Users. This includes refining features and improving metrics interpretation guidance.

  • 3. Competitors

    In the second and third months, RunWize will conduct a high-level competitor analysis to understand its unique selling points and identify areas for improvement, thereby enhancing its feature development and marketing strategies.

  • 4. New Features

    From the third to fifth month, we'll focus on developing new features to enhance the app's appeal, including community features, advanced training programs, customisable workout plans, enhanced social sharing, and privacy settings, aiming to attract and retain users.

  • 5. Outreach

    The final phase, from the fourth to the sixth month, will focus on strategic marketing, launching targeted campaigns, collaborating with fitness influencers, and developing loyalty programs to enhance app visibility, user engagement, and subscription growth, while also analysing marketing success.
